## Screen capture list (replace placeholders when available)
Capture these screens to show real product depth. Suggested views and focus areas:
1. Composer -- show base post + per-platform variants, validation panel, and preview tabs.
2. Calendar -- week view with drag/drop cards and a reschedule modal.
3. Media library -- grid view with metadata and preview drawer.
4. Channels -- connected accounts, permission status, and reconnect hints.
5. Logs -- published + failed jobs with retry controls.
6. Settings -- AI provider keys (BYO model) and workspace settings.

Recommended capture notes:
- Use a populated workspace name and 6-10 scheduled posts across 2-3 platforms.
- Include at least one warning state (validation or media constraints).
- Capture both light and dark themes if available.
- Use 2x resolution and export WebP + PNG.

## Format guidance (industry standard)
- Export masters in 1:1 and 16:9; optionally 4:5 for mobile-heavy feeds.
- Keep text safe margins (10-12% padding) for platform cropping.
- Provide WebP for web use and PNG for fallback.
- Target file size under ~2-3 MB per image to avoid compression artifacts.
