# Agent Instructions

## Public Site Copy

- All public-facing copy must use standard British English spelling, grammar, and punctuation.
- Prefer British spellings such as "optimise", "organisation", "authorise", "behaviour", "centre", "licence" as a noun, "practise" as a verb, and "artefact".
- Use "licence" for the noun and "license" for the verb only when that distinction matters.
- Do not change code identifiers, CSS properties, third-party names, API fields, package names, or external product terminology solely to enforce British spelling.

## Flat18 Ease of Communication Standard

Apply the Flat18 Ease of Communication Standard (F18 EoCS) to website copy, legal explainers, service pages, forms, CTAs, metadata, and schema descriptions:

- Use plain English. Prefer everyday words over inflated or obscure terms.
- Be clear, direct, and specific. Say what happens, who does it, and what the reader should expect.
- Use active voice where practical.
- Keep sentences and paragraphs short enough to scan.
- Use headings, lists, and examples to make complex information easier to understand.
- Avoid unnecessary jargon. If a technical term is needed, explain it in context.
- Use a positive, helpful tone without exaggeration or sales fluff.
- Contractions are allowed and often preferred when they make copy sound natural, for example "we're", "we'll", "you'll", "it's", and "don't".
- Legal and policy copy should remain accurate and careful, but it should still be readable and human.
- When editing visible copy, check nearby metadata, schema descriptions, alt text, and CTA text for the same language standard.
